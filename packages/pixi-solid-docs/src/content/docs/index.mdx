### At a glance

Pixi Solid is a library that provides [SolidJS](https://docs.solidjs.com/) components and hooks for working with [PixiJS](https://pixijs.com/).

- 📦 **Full component coverage** — Every major PixiJS display object has a corresponding component.
- ⚡ **Signals-driven reactivity** — State changes automatically update your scene.
- 🧹 **Automatic cleanup** — Components clean up after themselves on unmount.
- 🧪 **Testable without a browser** — Context, hooks, and ticker are built for simulation.

---

### Install

```bash
npm i pixi-solid
```

Peer dependencies of

```json
{
  "pixi.js": "^8.14.3",
  "solid-js": "^1.9.10"
}
```

### Why pixi-solid?

```tsx
import { PixiCanvas, Sprite } from "pixi-solid";
import { Texture } from "pixi.js";

export const App = () => (
  <PixiCanvas style={{ width: "100%", height: "100vh" }} background="#1099bb">
    <Sprite texture={Texture.WHITE} x={100} y={100} />
  </PixiCanvas>
);
```

<dl>
  <dt>Declarative scene graphs</dt>
  <dd>Compose PixiJS objects with JSX instead of imperative addChild/removeChild calls.
  The tree is your scene.</dd>

  <dt>Automatic lifecycle</dt>
  <dd>Components clean up after themselves on unmount: display objects, event listeners,
  ticker subscriptions, textures. No manual disposal tracking.</dd>

  <dt>Shared reactivity</dt>
  <dd>Signals and stores drive both canvas content and HTML UI from the same state.
  No bridging layer or two-way sync required.</dd>

  <dt>Unified timing</dt>
  <dd>Animations, frame callbacks, and sprite updates all synchronise to the same ticker
  context. No timing drift between onTick, useSpring, or AnimatedSprite.</dd>

  <dt>Full PixiJS coverage</dt>
  <dd>Props, refs, event handlers, and compatibility with plain PixiJS children if you
  need it.</dd>

  <dt>Testable by design</dt>
  <dd>Context providers, hooks, and the ticker are structured for simulation in tests.
  No browser or canvas required for unit tests.</dd>
</dl>
